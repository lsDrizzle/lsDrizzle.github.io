Kernel Method

Linear classification algorithms cannot deal with samples that cannot be divided by a line. In order to solve this problem, the perceptron learning algorithm (PLA) evolves to the multi-layer perceptron, i.e., neural networks. Another solution is addressing samples by some sort of nonlinear transformation ϕ(x)\phi(x)ϕ(x), so that the dimension of samples is increased. This is the core of the Cover Theorem: high-dim samples are more likely to be divided linearly than low-dim samples.

Recall the Hard-margin SVM. We can see there is an inner product term in the dual problem: xiTxjx_i^Tx_jxiT​xj​, which means we need to compute the inner product frequently, and this step is replaced by computing ϕ(xi)Tϕ(xj)\phi(x_i)^T\phi(x_j)ϕ(xi​)Tϕ(xj​) after the introduction of nonlinear transformation. Unfortunately, due to the strongly high dimension of ϕ(x)\phi(x)ϕ(x), it’s much harder to computing ϕ(x)\phi(x)ϕ(x) and even ϕ(xi)Tϕ(xj)\phi(x_i)^T\phi(x_j)ϕ(xi​)Tϕ(xj​) through the superficial formula sometimes.

Actually we only care about the result of ϕ(xi)Tϕ(xj)\phi(x_i)^T\phi(x_j)ϕ(xi​)Tϕ(xj​), so here comes the definition of Kernel Function, and we can use it to evade large amount of computations.

If for any x,x′∈Xx, x'\in\mathcal{X}x,x′∈X, there exists ϕ:X→Z\phi: \mathcal{X}\rightarrow Zϕ:X→Z such that K(x,x′)=ϕ(xi)Tϕ(xj)=<ϕ(xi),ϕ(xj)>K(x, x')=\phi(x_i)^T\phi(x_j)=<\phi(x_i),\phi(x_j)>K(x,x′)=ϕ(xi​)Tϕ(xj​)=<ϕ(xi​),ϕ(xj​)>, then we call K(x,x′)K(x,x')K(x,x′) a kernel function. It is obvious that K:X×X→RK: \mathcal{X}\times\mathcal{X}\rightarrow\RK:X×X→R and K(x,x′)=K(x′,x)K(x,x')=K(x',x)K(x,x′)=K(x′,x).

Positive Definite Kernel Function

For a positive definite kernel function, ϕ∈H\phi\in\mathcal{H}ϕ∈H, which is the Hilbert Space. The necessary and sufficient condition of it is that for any x1,x2,...xN∈Xx_1,x_2,...x_N\in\mathcal{X}x1​,x2​,...xN​∈X, the corresponding Gram Matrix K=(K(xi,xj))K=(K(x_i,x_j))K=(K(xi​,xj​)) is positive semidefinite.
